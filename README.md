# 淘天招聘职位爬取与筛选系统

一个完整的招聘信息爬取和智能筛选工具，支持一键爬取淘天集团招聘官网职位数据，提供自然语言交互式筛选和简历匹配功能。

## 系统要求
- Python 3.7+
- 浏览器：Chromium (自动安装)

---

## 🚀 第一步：爬取职位数据并构建数据库

### 1. 安装依赖
```bash
pip install -r requirements.txt
playwright install chromium
```

### 2. 运行爬虫抓取职位
使用 `main.py` 的 `crawl` 命令爬取淘天招聘官网数据：

```bash
# 基础用法：爬取前10页社会招聘研发类职位（默认）
python main.py crawl

# 自定义爬取页数（例如爬取20页）
python main.py crawl --pages 20

# 无头模式（后台运行，不显示浏览器窗口）
python main.py crawl --pages 20 --headless

# 爬取全量职位（不限制研发类，包含所有类别）
python main.py crawl --pages 50 --all-jobs
```

爬虫会自动将所有抓取到的职位信息存入SQLite数据库中。

### 3. 查看数据库状态
爬取完成后，可以查看数据库统计信息：
```bash
python main.py stats
```
会显示当前数据库中存储的职位总数。

## 🔍 第二步：筛选匹配职位信息

系统提供两种使用方式，按需选择：

### 方式一：自然语言交互式筛选（推荐）
直接用中文描述你的筛选需求，系统自动解析并匹配职位：

```bash
python job_filter_nlp.py
```

进入交互模式后，你可以直接输入各种筛选指令：

#### 常用指令示例：
```
# 关键词筛选
要算法、大模型、对话系统相关的职位
不要广告、视觉、多模态方向的

# 地点筛选
只要北京和杭州的职位
上海、深圳、广州的工作

# 经验筛选
3年以上工作经验
不超过5年经验的

# 组合筛选
要算法相关，北京的，3年以上经验，不要广告部门的

# 结果操作
按经验从高到低排序
显示第2页，每页20条
导出到result.csv
重置筛选条件
退出
```

## 📁 文件说明
| 文件名 | 功能说明 |
|--------|----------|
| `main.py` | 系统主入口，提供爬取、简历匹配、统计等命令行功能 |
| `job_filter_nlp.py` | 自然语言交互式筛选工具 |
| `spider.py` | 淘天招聘网站爬虫模块 |
| `database.py` | SQLite数据库操作模块 |
| `matcher_simple.py` | 职位匹配算法模块 |
| `dump_jobs.py` | 数据库导出工具，可导出为JSON Lines格式 |
| `jobs.db` | SQLite数据库文件（爬虫运行后自动生成） |

## 🌐 方式三：BOSS直聘风格Web UI（最新功能）

我们提供了可视化的Web界面，仿照BOSS直聘的交互风格，支持标签勾选和实时职位推荐。

### 快速启动
```bash
# 一键启动前后端服务
./run.sh
```

启动后访问：
- 前端界面: http://localhost:3000
- 后端API文档: http://localhost:8000/docs

### 功能特性
1. **三栏式布局**：左侧标签选择、中间职位列表、右侧职位详情
2. **多维度标签筛选**：
   - 技能标签（Python、大模型、机器学习等20个高频技能）
   - 地点标签（支持多城市选择）
   - 工作经验筛选
   - 职位类型筛选
3. **智能匹配算法**：
   - 同类型标签：OR逻辑
   - 不同类型标签：AND逻辑
   - 综合得分：关键词匹配(50%) + 文本相似度(50%)
4. **职位列表**：显示匹配度、薪资、地点、经验等信息，支持按匹配度/薪资/发布时间排序
5. **职位详情**：查看完整的岗位职责、任职要求，支持一键跳转到原始职位页面

### Docker部署
```bash
# 构建并启动容器
docker-compose up -d --build

# 停止服务
docker-compose down
```

部署完成后直接访问 http://localhost 即可使用。

---

## 💡 使用提示
1. 首次使用请先运行爬虫抓取数据，否则筛选功能没有数据
2. 自然语言筛选支持任意组合的筛选条件，尽可能用自然的中文描述即可
3. Web UI需要先运行爬虫构建数据库后才能正常显示职位数据
